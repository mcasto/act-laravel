### **Real-Time vs Periodic Sync**  

- **Real-Time Sync** (Webhook-based)  
  - **Example:** A donor makes a contribution ‚Üí Mailchimp sends a **webhook** ‚Üí Laravel updates local DB and sends a thank-you email.  
  - **Best for:** Keeping everything instantly updated.  
  - **Downside:** Can require more API calls if you‚Äôre syncing a lot of data.  

- **Periodic Sync** (Scheduled Updates)  
  - **Example:** A Laravel **cron job** runs every 30 minutes to check for new members in Mailchimp and update your database.  
  - **Best for:** Reducing API calls.  
  - **Downside:** Not always up-to-date immediately.  

For your case:  
‚úÖ **Use webhooks** for instant updates (like ticket purchases triggering show reminders).  
‚úÖ **Use periodic sync** for admin tools (like updating the local database with Mailchimp members once a day).
